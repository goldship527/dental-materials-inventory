# 公開デモ自己点検チェックリスト

対象: `dental-materials-inventory` 公開デモ
使い方: デモ画面を1画面ずつ開き、本リストの該当項目を上から順に確認。OK/NGをチェックし、NGはメモ欄に状況を書き残す。
所要時間目安: 約60〜90分(全画面通し)

---

## 0. 事前準備

| | 確認項目 | OK/NG | メモ |
|---|---|---|---|
| 0-1 | デモURLにPC(通常画面幅)で開ける | | |
| 0-2 | 同じURLをタブレット幅(768〜1024px)で開ける(PCのウィンドウを狭めるかDevToolsの端末エミュレーション) | | |
| 0-3 | 同じURLをスマホ幅(375〜414px)で開ける | | |
| 0-4 | Vercel SSO画面が挟まらない(挟まる場合は販売前に必ず外す) | | |
| 0-5 | ブラウザのURLが `*.vercel.app` の自動URLか、独自ドメインか確認 | | |

## 20. 終わったあとに

すべての確認が終わったら、以下を整理して開発側に共有すると効率的です。

- NGがついた行のID(例: 11-9, 14-13, 17b-9)と現象のメモ
- 「これは仕様通りNGだが販売前に追加すべき」と「これはバグ」を区別してメモする
- スマホ・タブレット・PCのそれぞれで撮ったスクリーンショット(画面遷移ごとに2〜3枚)
- 印刷プレビューのスクリーンショット(/shortage と /orders それぞれ1枚)

NGが多い項目(複数の画面で同じ問題が出るもの、例: 二重送信防止が全画面で甘い、競合検出が全画面で甘い、など)は、個別画面のバグではなく**設計レベルの直し**になります。前回お渡ししたレビューレポートの「優先して直すべき順番」と突き合わせて、対応順を決めてください。
