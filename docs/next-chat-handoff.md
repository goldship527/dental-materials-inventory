## 2. 守るルール

- 実在製品データ、患者情報、実在クリニック名、秘密情報をコード、seed、テスト固定値、Gitに入れない
- DB接続文字列、Supabaseパスワード、Vercel環境変数の実値をチャットやドキュメントに書かない
- Prisma 6系固定
- Tailwind CSSを使う
- shadcn/uiは未導入方針を維持する
- バーコード読み取りだけでは在庫数、発注候補、在庫変更履歴を変更しない
- 削除、外部発注送信、メール送信、発注書PDF生成、納品確認は追加しない
- 変更後は `docs/dev-log.md` に作業内容、判断、セキュリティメモ、検証を追記する
- 検証は最低限 `corepack pnpm typecheck` と `corepack pnpm build` を通す

## 5. 直近で話した本番導入・料金の考え方

本番導入では、Vercel / Supabaseは有料前提で考える。

概算:

```text
Vercel Pro      約 $20/月 から
Supabase Pro   約 $25/月 から
合計目安        約 $45/月 から + 使用量 + 税 + 追加機能
```

これはインフラ原価に近い。客先への提供価格は、導入支援、データ投入、操作説明、保守、問い合わせ対応、バックアップ確認を含めて考える。

5クリニック1法人で扱う場合のたたき台:

```text
初期導入費: 20万〜30万円程度
月額利用料: 3万〜5万円程度
対象: 1法人・5クリニック程度まで
```

別案:

```text
法人基本料: 月20,000円
クリニック追加: 1拠点あたり月5,000円
5クリニックの場合: 月45,000円
```

これは正式見積ではなく、料金設計のたたき台。
